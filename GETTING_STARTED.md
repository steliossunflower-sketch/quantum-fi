# Getting Started Guide

Welcome! This guide will help you get started with sharing your Golden Ratio Quantum Pulse discovery.

## Step 1: Set Up Your Accounts (Free!)

### GitHub Account
1. Go to [github.com](https://github.com)
2. Click "Sign up"
3. Choose a username (this will be your profile URL)
4. Verify your email
5. **Done!** You now have a GitHub account

### Medium Account
1. Go to [medium.com](https://medium.com)
2. Click "Get started" or "Sign in"
3. You can sign up with Google, Facebook, or email
4. **Done!** You can now write articles

### Dev.to Account
1. Go to [dev.to](https://dev.to)
2. Click "Create account"
3. Sign up with GitHub (easiest!) or email
4. **Done!** Another platform to share your work

### YouTube Channel
1. Go to [youtube.com](https://youtube.com)
2. Sign in with your Google account (or create one)
3. Go to YouTube Studio
4. Click "Create" > "Upload videos"
5. **Done!** You now have a YouTube channel

---

## Step 2: Upload Your Code to GitHub

### Option A: Using GitHub Website (Easiest for Beginners)

1. **Create a new repository:**
   - Go to github.com
   - Click the "+" icon in the top right
   - Click "New repository"
   - Name it: `quantum-fi` or `golden-ratio-pulse`
   - Make it Public
   - Click "Create repository"

2. **Upload your files:**
   - On the new repository page, click "uploading an existing file"
   - Drag and drop these files:
     - `pulse_comparison.py`
     - `README.md`
     - `LICENSE`
     - `.gitignore`
     - `requirements.txt`
     - `blog_post.md`
     - `video_script.md`
   - Scroll down, write a commit message: "Initial commit: Golden Ratio Pulse discovery"
   - Click "Commit changes"

3. **Done!** Your code is now on GitHub!

### Option B: Using Git Command Line (If You Want to Learn)

1. **Install Git:**
   - Download from [git-scm.com](https://git-scm.com)
   - Install it (use default settings)

2. **Open Terminal/Command Prompt:**
   - Navigate to your project folder:
     ```bash
     cd "C:\Users\User\Documents\quantum fi"
     ```

3. **Initialize Git:**
   ```bash
   git init
   git add .
   git commit -m "Initial commit: Golden Ratio Pulse discovery"
   ```

4. **Connect to GitHub:**
   - Create repository on GitHub (same as Option A, step 1)
   - Copy the repository URL
   - Run:
     ```bash
     git remote add origin https://github.com/YOURUSERNAME/YOURREPO.git
     git branch -M main
     git push -u origin main
     ```

## Step 4: Deploy Your Website

### Using GitHub Pages:

1. **Move website files to repository root:**
   - Copy `website/index.html` and `website/styles.css` to your repository
   - Or keep them in the `website` folder

2. **Enable GitHub Pages:**
   - Go to your repository on GitHub
   - Click "Settings"
   - Scroll to "Pages" (left sidebar)
   - Under "Source", select "Deploy from a branch"
   - Choose branch: "main"
   - Choose folder: "/ (root)" or "/website"
   - Click "Save"

3. **Your website will be live at:**
   `https://YOURUSERNAME.github.io/YOURREPO/`

4. **Update links in `index.html`:**
   - Replace `yourusername` with your actual GitHub username
   - Update the GitHub repository URL

## Step 5: Create Your YouTube Video

1. **Prepare:**
   - Read through `video_script.md`
   - Practice explaining your discovery
   - Have your code ready to show

2. **Record:**
   - Use screen recording software:
     - Windows: Built-in Game Bar (Win+G) or OBS (free)
     - Mac: QuickTime (built-in)
   - Record yourself explaining while showing the code
   - Show the plots and results

3. **Edit:**
   - Use free software:
     - DaVinci Resolve (free, professional)
     - OpenShot (free, simple)
     - Canva (free, web-based)
   - Add captions, text overlays, background music

4. **Upload:**
   - Go to YouTube Studio
   - Click "Create" > "Upload video"
   - Add title, description, tags
   - Set thumbnail
   - Publish!

## Troubleshooting

### "I don't understand Git"
- That's okay! Use Option A (GitHub website) instead
- You can learn Git later if you want

### "My website isn't showing up"
- Wait 5-10 minutes after enabling GitHub Pages
- Check that your files are in the right folder
- Make sure the branch is "main" or "master"

### "I'm scared to share my work"
- Remember: everyone starts somewhere!
- Your discovery is real and valuable
- The worst that can happen is no one sees it (but they will!)

### "I don't have time for all of this"
- Start with just one thing (GitHub is easiest)
- Do the rest when you have time
- Even one platform is better than none!
